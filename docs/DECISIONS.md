# Технические решения

---

## 2026-07-14 — Спринт 0

### 1. CMS / стек

**Решение:** Статика + YAML + vanilla JS

**Почему:**
- Легко копировать и перезапускать (нет БД)
- Заказчик редактирует `config/*.yaml` на GitHub без разработчика
- Быстрая загрузка, минимум зависимостей

### 2. Хостинг

**Решение:** GitHub Pages (`stiapanreha-dev/fuel-landing`)

**URL:** https://stiapanreha-dev.github.io/fuel-landing/

**Почему:** уже есть аккаунт и gh-cli, аналогично `translator-landing`.

### 3. Загрузка конфигов

**Решение:** `fetch()` + js-yaml (CDN) в браузере

**Почему:** без build-шага; правка YAML на GitHub → сразу на сайте.

### 4. Обработчик форм

**Решение:** отложено на спринт 5

**Ограничение:** GitHub Pages — только статика, нет PHP/Node на сервере.

**План:** Formspree / Web3Forms / Cloudflare Worker / перенос форм на VPS при необходимости.

### 5. Изображения

**Решение:** SVG-заглушки в `images/`, пути в конфигах

**Статус:** логотип и фото от заказчика пока нет.
